# Skill2Scale Data Analytics Internship

## Folder & File Management SOP

## Purpose

This document defines the standard structure for organizing all files in the Skill2Scale Data Analytics Internship project. It ensures consistency, clarity, and a professional workflow across datasets, analysis, reporting, and dashboards.

## Folder Structure

All project files follow this structure:

```text
datasets/
  ├── raw/
  ├── cleaned/

01_week_1_data_foundations/
02_week_2_visualization_dashboard/
03_week_3_business_intelligence/
04_week_4_executive_reporting/

assets/
sql/
```

Each weekly folder contains:

* analysis files
* reports
* visuals
* dashboards

## File Naming Convention

All files must follow a consistent naming format:

**Format:**
project_description_version

**Examples:**

* campaign_data_cleaned_v1.xlsx
* roi_analysis_week1.xlsx
* marketing_dashboard_q3.png
* executive_report_final.pdf

**Do NOT use:**

* FINAL_final.xlsx
* newfile.xlsx
* untitled documents

## Data Management Rules

* Raw data must NOT be modified
* Cleaned datasets must be stored in `datasets/cleaned/`
* Version control must be used:

  * v1, v2, v3 OR
  * date-based naming

## Reporting Standards

All reports must:

* Be stored in relevant weekly folders
* Include summary, insights, and recommendations
* Be written in clear business language
* Be clearly titled and dated

## Visuals & Dashboards

* Export charts as PNG or PDF
* Store dashboards in `visuals/` or `dashboards/`
* Use descriptive file names

## Workflow Structure

The analytics process follows this order:

Raw Data → Cleaning → Analysis → Visualization → Reporting → Presentation

## Objective

This structure ensures:

* organized project workflow
* consistent file management
* professional reporting standards
* portfolio-ready documentation
